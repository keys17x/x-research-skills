# x推文研究转内容

## 一句话定位

把外部看到的一批英文分析推文，转成 **你自己能继续判断、写作和发帖** 的中文内容资产。

## 上下游关系

- 上游常见输入：
  - `x行情观点搜索`
  - `x高质量分析账号筛选`
- 上游依赖：`twitter-cli-browser`

把一批 X/Twitter 上的英文分析推文，转成**适合中文研究、判断、写作和发帖**的内容资产。

这个 skill 不是单纯翻译，而是：

- 先理解推文里的信息结构
- 再做去重、降噪、观点聚类
- 最后转成中文摘要、观点提炼、草稿或选题

## 适用场景

适合：
- 把英文分析推文快速转成中文研究材料
- 为发帖、评论、选题准备素材
- 把 `x行情观点搜索` 或优质账号池的结果转成内容资产
- 总结某个主题在 X 上的主流观点和分歧

不适合：
- 不筛选就直接批量翻译
- 直接复制别人的观点对外发布
- 把所有推文内容都当事实依据

## 推荐输入格式

输入通常包含：
- 一批推文样本
- 主题：`BTC` / `ETH` / 某叙事 / 某事件
- 输出用途：
  - 自己看
  - 中文观点摘要
  - 可发短帖草稿
  - thread 提纲
- 风格：
  - 客观
  - 稳健
  - 更有观点感
- 长度：
  - 极简
  - 标准
  - 详细

如果用户只说：
- “把这些推文整理成中文”
- “把这批推文转成我能发的内容”

默认做法：
- 先做中文研究摘要
- 再给一版更短的中文观点提炼
- 如内容适合公开表达，再补一版草稿

## 执行流程

### Step 1: 接收推文样本

样本来源通常有两种：
- `x行情观点搜索` 搜出来的一批推文
- `x高质量分析账号筛选` 后，从优质账号拉下来的内容

先确认：
- 主题是否一致
- 样本量是否足够
- 是否混入明显无关内容

### Step 2: 观点聚类

不要直接逐条翻译。先把推文分成常见观点桶，例如：
- 偏多
- 偏空
- 震荡等待确认
- ETH/BTC 相对强弱
- 宏观周期视角
- 短线技术分析
- 纯噪音 / 营销

目标是先看“结构”，再看“文字”。

### Step 3: 去重和降噪

优先剔除：
- 重复表达同一个意思的推文
- 明显营销/广告/引流
- 只有情绪，没有逻辑
- 与主题相关度很低的内容
- 只有结论没有推演过程、只给一个价位喊多喊空的内容，除非它能代表市场情绪，否则默认降权

优先保留：
- 有图表 / 数据 / 结构的内容
- 有信息增量的观点
- 能代表共识或分歧的样本

### Step 4: 提炼信息骨架

把样本压成几个核心问题，而不是围着原推文复述。典型骨架：
- 当前市场主流到底偏什么方向
- 核心争议点是什么
- 哪些价位 / 指标 / 结构被反复提及
- 哪些观点只是情绪，哪些更有增量

### Step 5: 生成多档中文产物

默认至少生成下面两层：

#### A. 研究摘要版
适合用户自己先看：
- 主流观点
- 分歧点
- 值得注意的逻辑
- 风险提醒

#### B. 中文观点提炼版
适合快速形成自己的判断：
- 市场现在在赌什么
- 哪些逻辑更值得继续看
- 哪些只是情绪噪音

如用户明确需要，再补：

#### C. 可发内容草稿版
形式可以是：
- 短帖
- 多条 thread 提纲
- 朋友圈式观点
- 选题框架

### Step 6: 加边界和风险提示

必须明确区分：
- “X 上别人怎么说”
- “现在已经被验证的事实”
- “适合公开表达的内容”

不要把别人的观点直接包装成用户自己的确定性结论。

## 推荐输出模板

### 模板1：研究摘要版

```markdown
### BTC 市场观点摘要
- 主流情绪：偏谨慎 / 略偏空 / 分歧明显
- 核心观察点：关键支撑、背离、ETH/BTC 相对强弱
- 市场主要分歧：是短线洗盘，还是结构转弱确认
- 值得继续跟踪：某位分析师提到的关键价位 / 结构
```

### 模板2：中文观点提炼版

```markdown
今天 X 上关于 BTC / ETH 的主流讨论大致有三类：
1. 认为 BTC 已出现背离，若失守关键位可能加速回撤
2. 认为短线仍有反抽空间，但还不足以确认全面转强
3. 认为 ETH 当前更大的问题在于相对 BTC 仍偏弱
```

### 模板3：可发短帖版

```markdown
看了一圈今天 X 上关于 BTC/ETH 的分析，整体情绪并不算乐观。
BTC 这边大家更关注背离和关键支撑，ETH 这边更多人在看 ETH/BTC 会不会继续走弱。
现在更像是在等确认，不太像单边一致看多。
```

## 质量标准

产出时尽量满足：
- 不是机械翻译，而是结构化重写
- 保留原始观点差异，不强行揉成单一结论
- 中文表达自然、简洁、可直接继续使用
- 公开表达版本要克制，不越界喊单

## 默认工作方式

如果用户没有特别说明，默认：
- 先做研究摘要版
- 再做中文观点提炼版
- 如内容确实适合公开表达，再补一版草稿
- 如果样本质量太差，要明确提示“噪音高，不建议直接产出公开内容”

## 风险与边界

- 这个 skill 用于**研究与内容准备**，不是照搬别人观点
- 推文里的判断可能互相冲突，要保留分歧，不强行统一
- 遇到明显营销话术，要降低权重或直接剔除
- 不要把短线图表分析包装成长期确定性结论
