# x高质量分析账号筛选

## 一句话定位

先从推文样本里找出 **以后该长期盯谁**，再把候选作者沉淀成可持续跟踪的研究账号池。

## 上下游关系

- 上游常见输入：`x行情观点搜索`
- 上游依赖：`twitter-cli-browser`
- 常见下游：`x推文研究转内容`

用 `twitter-cli-browser` 这条路线，从 X/Twitter 上围绕 **BTC / ETH / 某个赛道 / 某个叙事** 找出**长期值得盯**的高质量分析账号，并输出一个可持续跟踪的账号池。

这个 skill 的目标不是找“哪条推文火”，而是找：

- 哪些账号在持续输出相关主题
- 哪些账号是真分析，哪些只是噪音或营销
- 哪些账号值得纳入长期 watchlist

## 适用场景

适合：
- 给自己建立稳定的 X 研究账号池
- 找值得长期盯的分析师 / 研究账号
- 从推文样本沉淀到长期信息源
- 为后续内容研究和转写建立可靠上游

不适合：
- 只看粉丝数就下结论
- 把互动高误当研究质量高
- 一次性找爆文，而不是找长期信息源

## 推荐输入格式

常见输入：
- 标的：`BTC` / `ETH` / `SOL`
- 赛道：`DeFi` / `L2` / `AI agent`
- 关键词：`macro` / `onchain` / `chart`
- 目标数量：`5` / `10` / `20`
- 风格偏好：技术分析 / 宏观 / 链上 / 新闻 / 混合
- 时间范围：最近 `7d` / `30d`

如果用户说得很简单，比如：
- “帮我筛一批 BTC 分析账号”
- “看看 ETH 这边有哪些值得盯的人”

默认做法：
- 先从最近 `7d` 的分析推文样本开始
- 候选账号抓 `10-20` 个
- 最终筛出 `5-10` 个可跟踪账号

## 执行流程

### Step 1: 先抓相关推文样本

不要一上来就凭印象推荐账号。先围绕主题搜索相关推文，例如：

```bash
twitter search "BTC analysis" --max 10 --yaml
twitter search "BTC chart" --max 10 --yaml
twitter search "ETH analysis" --max 10 --yaml
twitter search "ETH/BTC" --max 10 --yaml
```

如果是赛道：

```bash
twitter search "DeFi analysis" --max 10 --yaml
twitter search "AI agent tokens" --max 10 --yaml
```

目的：
- 找到正在持续讨论这个主题的人
- 给后续候选账号池提供样本

### Step 2: 提取高频作者

从样本里提取作者，优先关注：
- 反复出现的作者
- 在该主题下多次输出的人
- 明显不是偶然路过的账号

不要因为一条高互动推文就直接认定账号高质量。

### Step 3: 查看候选账号近期内容

对候选账号进一步抽查其近期内容，例如：

```bash
twitter user <handle>
twitter user-posts <handle> --max 10 --yaml
twitter user-posts <handle> --max 20 --yaml
```

观察：
- 最近内容是否持续围绕某个领域
- 是真分析，还是杂乱转发
- 是否经常贴图表 / 数据 / 结构化观点
- 是否大量带链接、带群、卖课、引流

### Step 4: 做账号画像

给每个账号打简单标签，例如：
- 技术分析型
- 宏观周期型
- ETH/BTC 相对强弱型
- 链上数据型
- 新闻聚合型
- 营销号 / 噪音号

这一步的重点不是绝对精确，而是让账号池可用。

### Step 5: 做质量粗筛

#### 正向信号
- 长期围绕某个主题持续输出
- 有图表、结构、价位、逻辑链
- 不是只会喊涨喊跌
- 互动里有一定真实讨论
- 观点前后有连续性

#### 负向信号
- 高频广告 / 卖会员 / 带链接引流
- 机器人味很重
- 标题党、情绪化但无结构
- 只转发，不输出自己的判断
- 内容过度分散，没有明确领域
- 近期内容里混入明显 fee-refund、群组导流、Telegram 引流等商业化动作时，要明确降权，不因单条分析质量就直接列入 A 级

### Step 6: 分层输出 watchlist

把账号分三层：

#### A级：重点盯
- 长期有研究价值
- 建议纳入日常观察池

#### B级：可备选
- 某些时候有用
- 但稳定性一般或营销感略重

#### C级：不建议重点看
- 噪音太大
- 营销比例过高
- 研究价值偏低

### Step 7: 给跟踪建议

最终不只是给名单，还要给使用建议：
- 哪些适合每天看
- 哪些适合在大行情时看
- 哪些只适合作为补充或反向指标

## 推荐输出模板

```markdown
### BTC 高质量分析账号筛选结果
**样本范围**：最近7天
**候选账号**：18 个
**筛后保留**：6 个

**一句话结论**：
当前值得盯的账号主要集中在技术分析派和周期判断派，纯营销号比例偏高。

**A级推荐**
1. `@xxx`
   - 类型：技术分析
   - 特点：图表清晰，观点连续，少广告
   - 推荐理由：最近多次围绕关键支撑/阻力做连续跟踪

2. `@yyy`
   - 类型：宏观周期
   - 特点：偏大级别结构判断
   - 推荐理由：发得不算高频，但输出质量稳定

**B级备选**
- `@zzz`：观点有意思，但营销倾向略重
- `@aaa`：短线分析多，但稳定性一般

**不建议重点看**
- `@bbb`：广告/引流比例过高
- `@ccc`：情绪化表达多，缺少分析逻辑

**后续建议**
- 每天盯 A 级
- B 级只在关键行情时抽查
- 后续可把 A 级账号交给 `x推文研究转内容`
```

## 质量标准

产出时尽量满足：
- 不是单纯列用户名单，而是给出理由
- 明确说明账号风格和适用场景
- 把“高互动”与“高质量”分开判断
- 少用空泛形容，多给结构化判断

## 默认工作方式

如果用户没有特别说明，默认：
- 先抓 2-4 组相关搜索词
- 抽取候选账号 10-20 个
- 进一步检查其中较强的 5-10 个
- 输出 A/B/C 三层结果
- 如用户后续要做内容表达，建议衔接 `x推文研究转内容`

## 风险与边界

- 这是**非官方 X 访问路线**，可用性依赖浏览器登录态
- 推荐结果带主观筛选色彩，不是绝对排名
- 不要把“某次说对”当成“长期值得盯”
- 遇到营销号密集的赛道，要明确提示样本污染

## 与其他 skill 的关系

- 可由 `x行情观点搜索` 提供上游样本
- 筛出的 A 级账号可交给 `x推文研究转内容`

一句话：
**这个 skill 负责找“以后该长期盯谁”。**
